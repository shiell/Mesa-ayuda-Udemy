🧩 Estructura de base de datos
El sistema está compuesto por las siguientes tablas:

users → usuarios del sistema (clientes, agentes, admins)

departments → departamentos internos (TI, Soporte, etc.)

tickets → incidencias registradas por los usuarios

ticket_messages → respuestas y seguimiento del ticket

attachments_ticket → archivos adjuntos al ticket principal

attachments_messages → archivos dentro de mensajes

status, priority, categories → personalización del flujo

logs → registro de cambios por ticket

faqs → artículos de ayuda o conocimiento
